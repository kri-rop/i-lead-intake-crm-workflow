# Workflow Testing

## Test Objective

The objective of the testing was to verify that incoming business inquiries are processed correctly from email intake to CRM documentation and Gmail draft creation.

The tests focused on data extraction, decision logic, HubSpot processing, error handling and human review.

## Test Environment

The workflow was tested with:

1. n8n Cloud
2. Gmail
3. Claude API
4. HubSpot CRM
5. Fictional test data

No real customer data was used.

## Test Cases

### 1. New Relevant Contact

Input:

A business inquiry containing a contact name, company, contact email address and a specific request.

Expected result:

1. Claude extracts the relevant information
2. JSON processing succeeds
3. The email validation returns true
4. The CRM relevance check returns true
5. HubSpot does not find an existing contact
6. A new contact is created
7. Inquiry category and priority are stored
8. A Gmail response draft is created

Result:

Passed

### 2. Existing Relevant Contact

Input:

A new inquiry from an email address that already exists in HubSpot.

Expected result:

1. HubSpot finds the existing contact
2. No duplicate contact is created
3. The existing contact is updated
4. Inquiry category and priority are stored
5. A Gmail response draft is created

Result:

Passed

### 3. Private Email

Input:

A private or unrelated email.

Expected result:

1. Claude identifies the message as not relevant for the CRM
2. The CRM relevance check returns false
3. No HubSpot contact is created or updated
4. No Gmail response draft is created

Result:

Passed

### 4. Consulting Request

Input:

A business inquiry requesting support with process automation.

Expected result:

1. The inquiry is categorized as Consulting
2. The message is processed as CRM relevant
3. The category is stored correctly in HubSpot

Result:

Passed

### 5. Demo Request

Input:

A business inquiry requesting a product or service demonstration.

Expected result:

1. The inquiry is categorized as Demo
2. The category is stored correctly in HubSpot
3. A response draft is created

Result:

Passed

### 6. Project Inquiry

Input:

A detailed request for a possible automation project.

Expected result:

1. The inquiry is categorized as Project Inquiry
2. The contact is created or updated
3. The category is stored correctly
4. A response draft is created

Result:

Passed

### 7. High Priority Inquiry

Input:

A project inquiry containing a short deadline, an upcoming project start and a request for a fast response.

Expected result:

1. Claude assigns High priority
2. Claude provides a clear priority reasoning
3. The priority is stored correctly in HubSpot

Result:

Passed

### 8. Missing Contact Email Address

Input:

A relevant business inquiry without a contact email address in the message content.

Expected result:

1. Claude returns an empty email field
2. The email validation returns false
3. The workflow stops before HubSpot
4. No Gmail response draft is created

Result:

Passed

### 9. Invalid JSON Response

Input:

An intentionally invalid JSON value used to simulate an incorrect Claude response.

Expected result:

1. The JavaScript node catches the parsing error
2. The field `parsing_error` returns true
3. The JSON validation returns false
4. The workflow stops before HubSpot and Gmail

Result:

Passed

### 10. Correct HubSpot Category

Input:

A relevant inquiry with a clearly identifiable request type.

Expected result:

The corresponding inquiry category is stored in the HubSpot contact property.

Result:

Passed

### 11. Correct HubSpot Priority

Input:

A relevant inquiry containing clear urgency indicators.

Expected result:

The assigned priority and the priority reasoning are stored correctly in HubSpot.

Result:

Passed

### 12. Gmail Draft Creation

Input:

A complete and relevant business inquiry.

Expected result:

1. The recipient email address is correct
2. The subject contains the inquiry category
3. The response text is complete
4. Paragraph formatting is displayed correctly
5. The email remains a draft
6. The email is not sent automatically

Result:

Passed

### 13. Human Review

Input:

A response draft generated by the workflow.

Expected result:

A person can review the recipient, subject, content, commitments, deadlines and tone before sending the email.

Result:

Passed

## Test Summary

All defined test cases were completed successfully.

The tests confirmed that the workflow:

1. Processes relevant inquiries
2. Stops private or incomplete messages
3. Prevents duplicate HubSpot contacts
4. Stores category and priority information
5. Handles invalid JSON in a controlled way
6. Creates complete Gmail response drafts
7. Maintains human control before external communication

## Remaining Operational Tests

For operational deployment, additional validation should cover:

1. Claude API outages
2. HubSpot API outages
3. Gmail connection failures
4. API quota limits
5. Retry behavior
6. Alerting
7. Failed request reprocessing
8. Role and permission changes
